Post-implementation critique surfaced one concrete weakness: the first pass of the per-ICP store enumerated JSON state files alphabetically, which could leak stale old ICP files into summaries and lose configured ICP order. I tightened that by resolving the current configured ICP list first, filtering summary/history views through that ordering, and only falling back to raw state files when a folder has ICP state but no persisted `icp_profiles.json` yet. After that refinement, I reran the Instagram agent/CLI tests plus the manifest/docs sync checks and they still passed.
