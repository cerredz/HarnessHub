### 1a: Structural Survey

The repository is a Python package centered on `harnessiq/`, with `tests/` providing regression coverage and `artifacts/` holding generated repository reference documents such as `artifacts/file_index.md`.

The prompt subsystem relevant to this task is concentrated under `harnessiq/master_prompts/`:

- `harnessiq/master_prompts/prompts/*.json` are the source-of-truth bundled master prompt assets.
- `harnessiq/master_prompts/registry.py` auto-discovers prompt JSON files and exposes `key`, `title`, `description`, and `prompt`.
- `harnessiq/master_prompts/__init__.py` exports the public SDK helpers used by tests and CLI code.
- `harnessiq/cli/master_prompts/commands.py` surfaces the prompt catalog through `harnessiq prompts`.
- `tests/test_master_prompts.py` is the main regression suite for prompt registry behavior and prompt catalog structure.

Relevant repo conventions:

- `harnessiq/` is the authoritative runtime tree.
- Documentation-only changes are still covered by focused regression tests when the docs represent runtime inventory or behavioral contracts.
- Existing `memory/` task folders capture internalization, clarifications, tickets, quality notes, and critiques for prior prompt-related work.

### 1c: Assumption & Risk Inventory

Assumptions:

- The "master prompt directory" is `harnessiq/master_prompts/`, because that is the live package owning bundled prompt assets.
- "Master plans" is user-facing wording for the bundled master prompt catalog, not a request to rename the package or APIs.
- The catalog content should come from the current prompt metadata in `harnessiq/master_prompts/prompts/*.json`, which is the live inventory reflected by repository docs.

Ambiguities:

- The user did not ask for automation explicitly, but the request includes an ongoing maintenance expectation for future prompt additions.

Risks:

- A hand-written README can drift out of sync with the prompt JSON metadata as new prompts are added.
- Editing `artifacts/file_index.md` directly would conflict with the repository rule that it is generated by `scripts/sync_repo_docs.py`.

Mitigations:

- Add a regression test that fails if any bundled prompt title or description is missing from the README.
- Keep the change scoped to the prompt package README and prompt tests.
